# 06 채널 플레이북 — Illinois Tool Works (ITW)

## 채널 전략 개요

ITW의 채널 전략은 **"고객이 있는 곳으로 간다"** 는 현장 밀착 원칙을 따른다. 디지털 채널은 인지·탐색 지원에 집중하고, 실제 구매 전환은 Application Engineer 직접 접촉과 트레이드쇼 현장에서 발생하는 구조다. (추정)

---

## 채널 아키텍처

```
채널 생태계
├── Owned (소유 채널)
│   ├── 기업 웹사이트 (itw.com + 세그먼트별 사이트)
│   ├── 이메일 마케팅
│   ├── 제품 카탈로그·기술 문서 포털
│   └── IR 웹사이트
├── Earned (획득 채널)
│   ├── 트레이드 저널·산업 미디어 기고
│   ├── 업계 분석가 리포트
│   └── 고객 레퍼런스·구전
├── Paid (유료 채널)
│   ├── LinkedIn 타깃 광고
│   ├── Google Search (산업용 키워드)
│   └── 트레이드쇼 스폰서십
└── Partner (파트너 채널)
    ├── 유통 파트너·딜러 네트워크
    ├── OEM 파트너 공동 마케팅
    └── 산업 협회 활동
```

## 채널별 상세 플레이북

### 채널 1: 기업 웹사이트

| 항목 | 내용 |
|------|------|
| 주요 URL | itw.com (공식), 세그먼트별 마이크로사이트 |
| 주요 방문자 | 제품 탐색 엔지니어, 조달 담당자, 투자자 |
| 핵심 목표 | 제품 사양 제공, Application Engineer 상담 연결, IR 정보 제공 |
| 콘텐츠 우선순위 | 제품 데이터시트 > 케이스 스터디 > 기술 가이드 |
| CTA 계층 | 1st: "Application Engineer 상담" / 2nd: "데이터시트 다운로드" |
| SEO 전략 | 산업별 기술 키워드 ("automotive welding solutions", "commercial dishwasher") |
| 측정 KPI | 상담 신청 전환율, 데이터시트 다운로드 수, 세션 당 제품 페이지 뎁스 |

### 채널 2: LinkedIn

| 항목 | 내용 |
|------|------|
| 계정 구조 | ITW 본사 페이지 + 세그먼트별 서브페이지 (추정) |
| 팔로워 규모 | 약 15만~20만 (추정) |
| 주요 오디언스 | 산업 엔지니어, 구매 담당자, 투자자, 취업 지원자 |
| 콘텐츠 믹스 | 기술 인사이트 40%, 고객 사례 30%, 기업 뉴스 20%, 채용 10% (추정) |
| 최적 포맷 | 텍스트+단일 이미지, 캐러셀(기술 팁 시리즈), 비디오(30~90초) |
| 게시 빈도 | 주 3~4회 (추정) |
| 유료 캠페인 | 세그먼트 타깃(직무·산업·회사 규모) Message Ad + Sponsored Content |
| 측정 KPI | 도달, 참여율, 웹사이트 클릭, 상담 폼 전환 |

**콘텐츠 예시 포맷:**
```
[기술 팁 캐러셀]
슬라이드 1: "자동차 용접 스패터 40% 줄이는 법"
슬라이드 2~5: 파라미터 최적화 가이드
슬라이드 6: "Application Engineer에게 무료 공정 진단 받기 →"
```

### 채널 3: 트레이드쇼 & 산업 이벤트

| 항목 | 내용 |
|------|------|
| 전략적 중요도 | 최고 (구매 결정 전환이 가장 많이 발생하는 채널) |
| 주요 참가 이벤트 | FABTECH, NAFEM, Automotive Tech Expo, PACK EXPO (추정) |
| 부스 전략 | 제품 시연 우선, 기술 세미나 병행, Application Engineer 상주 |
| 사전 캠페인 | 이벤트 6주 전부터 LinkedIn + 이메일로 사전 미팅 신청 유도 |
| 현장 목표 | 자격 리드(Qualified Lead) 수집, 기술 시연, 경쟁사 정보 수집 |
| 사후 캠페인 | 72시간 내 개인화 팔로업 이메일 + Application Engineer 연결 |
| 측정 KPI | 수집 리드 수, 시연 참여자 수, 상담 전환율, 계약 체결 기간 |

### 채널 5: 유통 파트너 네트워크

| 항목 | 내용 |
|------|------|
| 파트너 유형 | 산업용 유통사, 전문 딜러, MRO 공급사 |

## 채널별 퍼널 매핑

| 퍼널 단계 | 주요 채널 | 콘텐츠 유형 | KPI |
|---------|---------|-----------|-----|
| 인지 (Awareness) | LinkedIn 광고, 트레이드 미디어, SEO | 기술 인사이트, 인포그래픽 | 도달, 노출 |
| 관심 (Interest) | 웹사이트, YouTube, 이메일 | 케이스 스터디, 비디오 시연 | 세션, 체류 시간 |
| 고려 (Consideration) | 웹사이트 폼, 이메일 육성 | 백서, ROI 계산기, 데이터시트 | 리드 수집, 다운로드 |
| 의도 (Intent) | Application Engineer 직접 접촉, 트레이드쇼 | 현장 시연, 맞춤 제안서 | 상담 건수, 제안 수락율 |
| 구매 (Purchase) | 직판 또는 유통 파트너 | 계약서, 기술 사양서 | 계약 체결, 매출 |
| 유지 (Retention) | 이메일, 서비스 포털, 직접 Account Manager | 유지보수 가이드, 업그레이드 알림 | 재구매율, NPS |

## Steal Sheet — 채널 플레이북 레이어

| 항목 | ITW 방식 | 벤치마크 가능 포인트 |
|------|---------|-------------------|
| 엔지니어 = 최고의 채널 | Application Engineer의 현장 방문이 최고 전환율 채널 | 기술 전문가를 세일즈 채널화하는 구조 |
| 트레이드쇼 역할 재정의 | 전시가 아닌 "기술 컨퍼런스 + 영업 클로징 장" | 이벤트를 퍼포먼스 채널로 측정·운영 |
| 퍼널 단계별 채널 분리 | 인지↔전환↔유지 채널을 분리 설계, KPI 각각 다르게 설정 | B2B 복잡 구매 여정의 채널 역할 명확화 |
