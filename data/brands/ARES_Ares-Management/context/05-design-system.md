# 05 — Design System
## Ares Management Corporation

---

## 1. Design Philosophy

Ares design communicates **institutional permanence**. Every visual choice — color, typeface, layout grid, data visualization — signals that this firm has been here for decades and intends to remain. The aesthetic avoids trend-chasing. It is not a fintech. It is not a startup. The design vocabulary is closer to a sovereign institution than a consumer brand.

The underlying design principle is **legibility under scrutiny**: every layout must survive a skeptical pension CIO reading it on a dual-monitor setup at 7am. Information hierarchy trumps visual decoration. White space is earned, not abundant.

- Avoid gradients in primary brand applications; reserved for digital backgrounds only

## 3. Typography

### Primary Typeface

**Serif: Georgia / Garamond class** — used for long-form documents, LP reports, annual letters
- Signals permanence, authority, tradition
- Consistent with how institutional finance has communicated for 50 years
- Alternative: Freight Text Pro (estimated, for digital-first applications)

### Secondary Typeface

**Sans-Serif: Helvetica Neue / Inter** — used for UI, digital platforms, data labels, presentations
- Clean legibility at small sizes
- Preferred for data tables, axis labels, call-out boxes
- Alternative: Source Sans Pro (estimated, for web/digital)

### Type Scale (Estimated)

| Element | Size | Weight | Usage |
|---|---|---|---|
| H1 Document Title | 32–40pt | Bold | Report covers, section headers |
| H2 Section Header | 24–28pt | Semibold | Major sections |
| H3 Subsection | 18–20pt | Semibold | Subsections, table titles |
| Body Copy | 10–12pt | Regular | LP reports, white papers |
| Data Labels | 8–10pt | Regular | Chart annotations, footnotes |
| Caption | 8pt | Regular/Italic | Source citations, disclaimers |
| Pull Quote | 18–22pt | Light | Featured statistics, key metrics |

### Typography Rules

- Line length capped at 75 characters for body copy (readability)
- Minimum 10pt for any printed body text
- Legal/regulatory disclaimers never below 8pt
- Heading hierarchy must be strict — no skipping levels
- Italics used only for fund names, titles of documents, foreign terms

## 4. Logo System

### Primary Logo
- Wordmark "ARES" in Ares Navy, sans-serif, tracked wide
- Submark "Management" in lighter weight, aligned beneath
- Minimum clear space: equal to the height of the "A" on all sides

### Logo Variants (Estimated)

| Variant | Use Case |
|---|---|
| Full horizontal lockup | Primary; letterhead, presentations |
| Stacked lockup | Square formats; social media profile |
| Navy on white | Primary digital application |
| White on navy | Dark background applications |
| Navy on gold | Premium / awards applications |
| Single-color black | Legal documents, monochrome print |

### Logo Rules

- Never stretch, rotate, or apply drop shadows to the logo
- Minimum digital size: 120px wide
- Minimum print size: 1 inch wide
- Never place logo on busy photographic backgrounds
- Never alter the typeface or letter-spacing of the wordmark

## 5. Iconography & Illustration

**Style:** Line-based, geometric, minimal. No illustrative characters or cartoons.

**Icon categories used:**
- Financial instruments (bond, equity, real asset symbols)
- Process flow arrows (deal lifecycle, underwriting steps)
- Data indicators (up/down arrows, percentage rings)
- Building blocks (capital structure layers visualized as stacked rectangles)

**Illustration rules:**
- Never use stock photography of people in suits shaking hands
- Preferred imagery: architectural photography (buildings, infrastructure), aerial industrial, data visualization graphics
- Human photography only in executive profiles and ESG/community contexts
- All photography must carry a sense of permanence and scale — no casual or candid shots

## 6. Layout & Grid System

### Document Grid (LP Reports / White Papers)

- Page size: US Letter (8.5" x 11") or A4 for European materials
- Margins: 0.75" all sides minimum
- Column grid: 2-column for body text; 3-column for data-heavy sections
- Gutter: 0.25" between columns
- Header zone: 0.75" reserved for logo + pagination
- Footer zone: 0.5" reserved for disclaimers + page numbers

### Presentation Grid (Pitch Decks / Conference Slides)

- Aspect ratio: 16:9
- Title safe zone: 10% margin inset from all edges
- Maximum text per slide: 6 lines of body copy (data slides exempt)
- Chart-to-text ratio: 60/40 preferred (more visual than verbal)
- Slide count for institutional pitch: 20–35 slides
- Slide count for advisor-facing: 12–20 slides

### Digital Grid (Website / Portal)

- Max content width: 1280px
- Side padding mobile: 16px
- Side padding desktop: 80px
- Section spacing: 80–120px between major content blocks
- Table cell minimum height: 48px

## 7. Data Visualization Standards

### Chart Types by Use Case

| Data Type | Preferred Chart | Avoid |
|---|---|---|
| Return comparison over time | Line chart (multi-series) | Pie chart |
| AUM growth | Vertical bar | Horizontal bar |
| Portfolio allocation | Horizontal bar with % labels | 3D pie |
| Loss rate trend | Line chart | Table alone |
| Capital structure breakdown | Stacked bar | Donut chart |
| IRR / MOIC distribution | Scatter plot | Bubble chart |
| Spread vs. benchmark | Dual-axis line | Area chart |

### Chart Style Rules

- All charts must include: title, axis labels, data source, date of data
- Y-axis always starts at zero unless explicitly annotated when truncated
- Grid lines: light grey (#E5E7EB), horizontal only
- Legend placement: above or to the right of chart, never below
- Data point labels only on key inflection points — avoid label clutter

## 9. Steal Sheets — Design System Layer

### Steal Sheet 1: The "Capital Structure Tower" Visual Language
**What Ares owns:** Unique ability to visualize where in the capital structure a given investment sits — senior secured, unitranche, mezzanine, equity.
**How to steal it:** Build a proprietary "capital stack" visual component — a vertical tower diagram showing debt layers, equity cushion, and Ares position within the structure. Use it in every product pitch. Make it so recognizable that allocators associate the visual format with Ares before reading the label.

### Steal Sheet 2: The "Cycle Timeline" Data Visualization
**What Ares owns:** 25+ years of credit data spanning five market dislocations.
**How to steal it:** Build a proprietary "credit cycle timeline" infographic — a horizontal timeline from 1997 to present, marking each dislocation with Ares AUM, loss rate, and spread environment at that moment. Publish it annually with updated data. Competitors with less history literally cannot replicate this.

### Steal Sheet 3: The "Navy + Gold" Premium Positioning Color Lock
**What Ares owns:** A color combination that is simultaneously traditional (navy = institutional) and premium (gold = excellence/reward).
**How to steal it:** Enforce strict color discipline across all touchpoints — presentations, reports, website, co-branded materials, conference booths. Reject any "modernization" that dilutes navy-gold in favor of trend colors (teal, coral, sage). The color lock IS the brand consistency signal at the institutional level.
