## 훅 공식 (Hook Formulas)

### 훅 유형 1 — 충격적 수치 선언
```
[신제품]은 [이전 세대] 대비 [배수]x 빠릅니다.
```
예시: "H100은 A100 대비 AI 학습에서 최대 30배 빠릅니다."
사용 채널: 제품 발표 키노트, 기술 블로그 첫 문장, 소셜 광고

### 훅 유형 2 — 카테고리 재정의 선언
```
[기존 카테고리]의 시대는 끝났습니다. [새 카테고리]의 시대가 시작됩니다.
```
예시: "CPU 컴퓨팅의 시대는 끝났습니다. 가속 컴퓨팅의 시대가 시작됩니다."
사용 채널: GTC 키노트 오프닝, 기업 광고, 투자자 발표

### 훅 유형 3 — 불가능을 가능으로
```
[이전에 불가능했던 것]이 이제 가능합니다. [제품/플랫폼]으로.
```
예시: "실시간 4K 레이트레이싱이 이제 가능합니다. RTX로."
사용 채널: 제품 출시 광고, YouTube 프리롤, 게임 파트너십 콘텐츠

### 훅 유형 4 — 사용자 성과 증언
```
[고객 이름/기관]은 NVIDIA [제품]으로 [구체적 성과]를 달성했습니다.
```
예시: "Microsoft는 NVIDIA H100 클러스터로 GPT-4 학습 시간을 X% 단축했습니다."
사용 채널: 고객 사례 발표, 영업 자료, 기업 블로그

## 톤 가이드

### 기업 / AI 인프라 채널 톤
- **자신감 있는 선언형**: 조건절 없이 단정한다. "~할 수 있습니다"가 아닌 "~합니다."
- **수치가 먼저**: 주장보다 데이터를 앞에 배치한다.
- **미래형 현재시제**: "우리는 AI 공장을 만들고 있습니다" → "AI 공장의 시대입니다."
- **기술 어휘 사용**: 독자를 과소평가하지 않는다. FLOPS, HBM, NVLink, TOPS를 설명 없이 사용한다.
- **스케일 강조**: "세계 최대", "역사상 최초", "가장 빠른"을 수치로 뒷받침한다.

### 게이밍 / 소비자 채널 톤
- **흥분과 감각**: 시각적 언어를 사용한다. "볼 수 없었던 세계", "느껴지는 차이".
- **커뮤니티 언어**: 게이머 용어(fps, 레이트레이싱, 프레임드롭)를 자연스럽게 사용한다.
- **경쟁심 자극**: "프로게이머가 선택한 이유", "타협 없는 성능".
- **캐주얼 직접 화법**: "여러분의 게임 경험을 바꿔드립니다."

### 개발자 채널 톤
- **동료 개발자 시점**: 마케팅 언어를 최소화하고 기술 정확성을 최우선한다.
- **코드와 예제 우선**: 설명보다 작동하는 코드 스니펫을 먼저 제시한다.
- **투명한 한계 인정**: 제한사항을 숨기지 않는다. 신뢰 형성의 핵심.
- **커뮤니티 기여 인정**: 오픈소스 기여자, 논문 저자, GTC 발표자를 적극 호명한다.

## 콘텐츠 금지사항

- **모호한 최상급 미지원**: "최고의 AI 칩" 단독 사용 — 반드시 벤치마크 출처를 병기할 것
- **경쟁사 직접 비방**: AMD/Intel을 이름 붙여 조롱하지 않는다. 수치가 말하게 한다.
- **과도한 미래 약속**: 로드맵 미공개 기술을 확정 사실처럼 서술하지 않는다 (SEC 규정 리스크)
- **게이밍 이미지로 기업 채널 오염**: 소비자 게이밍 광고 스타일을 엔터프라이즈 자료에 혼용 금지
- **수치 미출처**: 성능 수치는 반드시 테스트 환경, 비교 기준, 측정 날짜를 명시한다
- **지나친 AI 유토피아 서술**: "AI가 모든 문제를 해결한다"는 식의 과장은 규제 리스크 및 신뢰도 저하

### Steal Sheet — 마케터가 훔쳐갈 3가지

1. **"키노트를 콘텐츠 슈퍼볼로 운영하라."** 젠슨 황의 GTC 키노트는 연 1~2회이지만, 그 하나의 이벤트가 수천 건의 미디어 기사, 유튜브 분석 영상, LinkedIn 반응 게시물을 자동 생성한다. → 마케터 적용: 소규모 브랜드도 연 1회 '선언의 순간'을 설계하라. 그 이벤트 전후로 콘텐츠 캘린더를 역산해 배치하면 적은 제작 비용으로 최대 확산을 만들 수 있다.

2. **"개발자 블로그를 가장 신뢰받는 마케팅 채널로 만들어라."** NVIDIA Developer Blog는 순수 기술 글이지만, 이것이 NVIDIA 브랜드를 기술 권위자로 각인시키는 1순위 채널이다. → 적용: 마케팅 팀이 쓴 글이 아닌, 실제 엔지니어가 작성한 기술 블로그를 외부 배포하라. 마케팅 목적으로 작성된 느낌이 없을수록 더 잘 퍼진다.

3. **"훅에 수치를 넣으면 공유율이 오른다."** "30x faster"는 공유되지만 "훨씬 빠릅니다"는 공유되지 않는다. 모든 성능 클레임을 숫자로 바꾸는 습관 하나로 콘텐츠 도달률이 달라진다. → 적용: 콘텐츠 첫 문장에 구체적 수치 + 비교 기준을 넣는 것을 스타일 가이드로 규정화하라.
