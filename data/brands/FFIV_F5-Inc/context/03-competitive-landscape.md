# F5 Inc. (FFIV) — 경쟁 지형 분석

## 시장 정의 (추정)

F5가 경쟁하는 시장은 크게 세 레이어로 구분된다.
1. **애플리케이션 딜리버리 컨트롤러(ADC)** — 전통적 코어 시장
2. **웹 애플리케이션 방화벽(WAF) / API 보안** — 성장 전략 시장
3. **멀티클라우드 네트워킹 및 엣지 보안** — 미래 시장

---

## 배틀카드 1: vs. Citrix (NetScaler ADC)

### 경쟁 구도 요약

Citrix NetScaler(구 ADC)는 F5 BIG-IP와 가장 직접적으로 경쟁하는 ADC 레거시 강자다. 2023년 Cloud Software Group에 매각된 이후 불확실성이 커졌다.

### F5 강점 (공식 포지셔닝 기반)

| 영역 | F5 | Citrix NetScaler |
|------|-----|-----------------|
| 멀티클라우드 지원 | 네이티브 멀티클라우드 정책 통합 | SaaS 전환 중, 일관성 부족 |
| 보안 통합 | WAF·DDoS·Bot 방어 내장 | 별도 라이선스 추가 필요 |
| 기업 안정성 | NASDAQ 상장 독립 기업 | 사모펀드 소유, 로드맵 불투명 |
| API 보안 | F5 Distributed Cloud로 통합 | 제한적 |

### 반박 스크립트 (추정)

> "NetScaler가 더 저렴하지 않나요?"
> → "단순 라이선스 비용은 비슷해 보이지만, WAF·Bot 방어를 추가하면 TCO가 역전됩니다. 당사 고객 사례를 보면 3년 TCO 기준 평균 23% 절감을 확인했습니다."

### 탈취 시그널

- Citrix 지원 계약 만료 6개월 전
- 멀티클라우드 전환 프로젝트 발주
- Citrix 라이선스 모델 변경 불만 표출

## 배틀카드 2: vs. Cloudflare

### 경쟁 구도 요약

Cloudflare는 엣지 보안·CDN·Zero Trust 접근제어 영역에서 F5 Distributed Cloud와 정면 충돌한다. 개발자 친화적 이미지와 셀프서비스 모델로 Mid-Market을 빠르게 침식 중이다.

### F5 강점 (공식 포지셔닝 기반)

| 영역 | F5 | Cloudflare |
|------|-----|-----------|
| 온프레미스 통합 | 하이브리드·온프레미스 완전 지원 | 클라우드 퍼스트, 온프레미스 미약 |
| 엔터프라이즈 컴플라이언스 | FedRAMP, PCI-DSS, HIPAA 인증 보유 | 일부 인증 진행 중 |
| 트래픽 제어 세분화 | Layer 7 정책 세밀도 최상급 | 규칙 복잡도 한계 |
| 기존 투자 보호 | BIG-IP 자산 연계 마이그레이션 | 완전 재구축 요구 |

### 반박 스크립트 (추정)

> "Cloudflare가 더 빠르고 저렴하지 않나요?"
> → "Cloudflare는 퍼블릭 웹에 최적화되어 있습니다. 규제 산업의 온프레미스 워크로드, 레거시 앱, 세밀한 L7 정책이 필요한 환경에서는 F5가 유일한 선택입니다."

### 탈취 시그널

- Cloudflare PoC 실패 후 엔터프라이즈 재검토
- 금융·공공 컴플라이언스 감사 대비 프로젝트
- 하이브리드 클라우드 마이그레이션 계획 발표

## 배틀카드 3: vs. Palo Alto Networks (Prisma Cloud + NGFW)

### 경쟁 구도 요약

Palo Alto는 NGFW에서 출발해 SASE·CNAPP으로 확장하며 F5의 보안 영역을 침범하고 있다. 단일 플랫폼 통합을 내세우는 Palo Alto의 "platformization" 전략이 CIO 레벨에서 매력적으로 작동한다.

### F5 강점 (공식 포지셔닝 기반)

| 영역 | F5 | Palo Alto |
|------|-----|----------|
| 애플리케이션 레이어 전문성 | Layer 7 딥인스펙션, 앱 인식 최상급 | 네트워크 레이어 강점, 앱 레이어 보완 중 |
| ADC 통합 | 로드밸런싱·보안 네이티브 통합 | 별도 솔루션 조합 필요 |
| 통신사 레퍼런스 | 글로벌 Top 10 통신사 대부분 고객 | 통신사 침투 제한적 |
| 가격 모델 | 워크로드 기반 유연 과금 | 플랫폼 번들, 초기 비용 높음 |

### 반박 스크립트 (추정)

> "Palo Alto 하나로 다 되지 않나요?"
> → "Palo Alto는 훌륭한 NGFW입니다. 하지만 애플리케이션 딜리버리, ADC, API 보안을 동시에 운영해야 한다면, F5가 없으면 별도 솔루션을 3개 더 붙여야 합니다."

### 탈취 시그널

- Palo Alto Prisma Cloud 도입 검토 중 ADC 공백 발견
- 기존 BIG-IP 환경에서 보안 레이어 추가 필요
- 멀티벤더 피로도 상승, 통합 필요성 대두
