# 02. 오디언스 맵 — Microsoft (MSFT)

## ICP (Ideal Customer Profile)

| 항목 | 내용 |
|------|------|
| 기업 규모 | 직원 500명 이상 엔터프라이즈, 또는 성장 단계 스타트업(Azure 스타트업 프로그램) |
| 산업군 | 금융·의료·제조·공공·리테일 (규제 산업 비중 높음) |
| 지역 | 글로벌 (북미 매출 비중 약 50%, 상업용 클라우드 성장 주도) |
| 의사결정자 | CIO, CISO, CTO, IT 구매위원회 |
| 기술 성숙도 | 기존 Microsoft 스택(Windows, Office, Active Directory) 운영 기업 |
| 핵심 페인포인트 | 레거시 시스템 현대화, 보안·컴플라이언스 부담, AI 도입 속도, IT 비용 최적화 |
| 구매 트리거 | 클라우드 마이그레이션 프로젝트, AI 전략 수립, 하이브리드 워크 정책 전환 |
| 연간 계약 규모 | 엔터프라이즈 Agreement 기준 연 수십만~수백만 달러 |

Microsoft의 ICP는 단순히 "큰 기업"이 아니라 "이미 Microsoft 생태계 안에 있는 기업"이다. Windows + Active Directory + Exchange 운영 기업은 Azure와 Microsoft 365로의 전환 마찰이 낮다. 이 잠금(lock-in) 구조가 ICP의 핵심 선택 기준이다.

### 페르소나 1 — 박지성, 엔터프라이즈 CIO (51세, 금융그룹 IT 총괄)

**배경**
대형 금융지주사 CIO. 300명 규모 IT 조직을 총괄하며 레거시 온프레미스 인프라의 클라우드 전환을 5년째 추진 중. 이사회에 분기별 IT 비용·보안 지표를 보고한다.

**핵심 목표**
- 금융감독원 클라우드 보안 가이드라인 준수
- 레거시 코어뱅킹 시스템과 Azure 하이브리드 연동
- Copilot 도입으로 사내 생산성 KPI 달성

**페인포인트**
- "Azure가 금감원 요건을 정말 다 맞추는지 확신이 없다."
- "우리 팀이 AWS도 쓰고 싶다고 하는데, 멀티클라우드 관리 복잡성이 두렵다."
- "Copilot 도입 ROI를 이사회에 어떻게 숫자로 보여주나."

**정보 소비 채널**
Gartner 리포트, Microsoft Ignite 컨퍼런스, LinkedIn, 업계 CIO 포럼

**구매 결정 요인**
규제 준수 인증(ISMS-P, ISO 27001), 국내 사례 레퍼런스, 전담 엔터프라이즈 어카운트 팀 지원

**Microsoft와의 접점**
기존 EA(Enterprise Agreement) 보유, Windows Server + SQL Server 온프레미스 운영 중

### 페르소나 2 — 이수진, 스타트업 CTO (34세, SaaS 스타트업 공동창업자)

**배경**
시리즈 B 단계 B2B SaaS 스타트업 공동창업자 겸 CTO. 개발자 25명 팀을 이끌며 글로벌 시장 진출을 준비 중. GitHub Copilot 헤비유저.

**핵심 목표**
- Azure 스타트업 크레딧으로 인프라 비용 최소화
- GitHub Actions + Azure DevOps로 CI/CD 파이프라인 자동화
- 미국 SOC 2 인증 취득을 위한 보안 인프라 구축

**페인포인트**
- "AWS가 더 익숙한데 Azure로 넘어가야 할 이유를 팀에게 설명하기 어렵다."
- "Microsoft 영업팀이 엔터프라이즈 중심이라 스타트업은 소외감을 느낀다."
- "Azure 문서가 방대해서 필요한 걸 찾는 데 시간이 너무 걸린다."

**정보 소비 채널**
GitHub, Stack Overflow, Microsoft Learn, X(트위터), Dev.to

**구매 결정 요인**
GitHub Copilot 연동, 스타트업 크레딧 프로그램 규모, 개발자 커뮤니티 평판

**Microsoft와의 접점**
GitHub 유료 플랜, Visual Studio Code(무료), Azure 스타트업 프로그램 지원

### 페르소나 3 — 김민준, 지식노동자 개인 사용자 (29세, 컨설팅펌 시니어 애널리스트)

### Steal Sheet — 오디언스 맵

1. **"생태계 잠금을 온보딩으로"** — Microsoft가 Azure 무료 크레딧과 GitHub 무료 플랜으로 개발자를 생태계에 먼저 진입시키고, 이후 유료 업셀하는 PLG(Product-Led Growth) 구조를 훔쳐라. 한국 SaaS 전략: 핵심 기능 무료 제공 → 팀 기능(협업) 유료화 → 기업 기능(보안·관리) 엔터프라이즈 계약으로 단계적 전환.

2. **"MVP 프로그램 = 무료 영업군단"** — Microsoft MVP 프로그램은 전문가 3,000명이 자발적으로 Microsoft 제품을 전도하는 구조다. 인증·혜택·커뮤니티 소속감으로 동기부여. 한국 브랜드에서 훔칠 구조: "공식 파트너 인증" 프로그램을 만들되 금전 보상보다 지위·콘텐츠 지원으로 설계.

3. **"Anti-Persona를 명시적으로 정의하라"** — Microsoft는 오픈소스 개발자 커뮤니티의 거부감을 무시하지 않고, GitHub 인수 후 MIT 라이선스 확대와 오픈소스 기여로 적극 대응했다. Anti-Persona의 불만을 제품·메시지로 해소하려는 시도가 장기적으로 오디언스를 확장했다. 훔칠 원칙: Anti-Persona를 "영원히 안 된다"가 아닌 "아직 안 된다"로 취급하고 분기별 재검토.
