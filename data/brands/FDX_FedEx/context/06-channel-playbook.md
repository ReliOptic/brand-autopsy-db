# 06. 채널 플레이북 — 페덱스 (FedEx)

## 채널별 전략

### 채널 1 — FedEx.com (공식 웹사이트 + 셀프서비스 포털)

| 항목 | 내용 |
|------|------|
| 역할 | 배송 견적·예약·추적의 핵심 트랜잭션 허브. 브랜드 정보와 셀프서비스의 교차점 |
| 주요 형식 | 실시간 배송 추적, 온라인 견적 도구, FedEx Ship Manager, 서비스 소개 페이지, 고객 지원 |
| 타겟 | 모든 화주 세그먼트 (소규모 셀러~대기업 물류 담당자) |
| KPI | 월간 방문자 수, 견적 완료율, 추적 페이지 재방문율, 온라인 발송 전환율 |
| 주기 | 상시 (트랜잭션 채널, 24/7 운영) |
| 톤 | 명료·직접·기능 중심. "어디로 보내시겠습니까?" — 고객 목적 달성을 방해하지 않는 미니멀한 언어 |
| 금지 | 불필요한 마케팅 배너로 트랜잭션 흐름 방해, 보증 조건 소자 숨기기 |

### 채널 5 — 이메일 마케팅 (FedEx Delivery Manager + 기업 CRM)

| 항목 | 내용 |
|------|------|
| 역할 | 트랜잭션 알림(배송 현황)과 마케팅(서비스 업셀·캠페인)의 이중 역할 채널 |
| 주요 형식 | 배송 상태 알림, 픽업 준비 알림, 볼륨 할인 오퍼, 시즌 캠페인, 신규 서비스 안내 |
| 타겟 | 기존 고객 (트랜잭션) + 이메일 구독자 (마케팅) |
| KPI | 오픈율, 클릭율(CTR), 재발송 전환율, 이탈(Unsubscribe)율 |
| 주기 | 트랜잭션: 실시간 / 마케팅: 주 1–2회 (추정) |
| 톤 | 명확·즉각. 트랜잭션 이메일은 필요 정보만. 마케팅 이메일은 혜택 중심 직접 화법. |
| 금지 | 트랜잭션 이메일에 마케팅 메시지 과부하, 개인화 없는 대량 발송, 동의 없는 수신자 추가 |

## 채널 시너지 맵

```
시즌 캠페인 / 제품 출시 (씨앗)
    ├── FedEx.com: 랜딩 페이지 + 온라인 견적 CTA
    ├── 이메일: 기존 고객 타겟 오퍼 + 할인 코드
    ├── LinkedIn: 기업 고객 대상 인사이트 + 사례 아티클
    ├── YouTube: 서비스 소개 영상 + 고객 스토리
    ├── X: 실시간 캠페인 업데이트 + CS 응대
    └── 파트너 채널: 쇼피파이·아마존 앱 내 프로모션 배너
```

**시너지 원칙**: 단일 캠페인 메시지가 6개 채널에서 각 타겟에 맞는 형식으로 동시 전달된다. 채널 간 일관성이 브랜드 신뢰를 강화한다.

### Steal Sheet — 마케터가 훔쳐갈 3가지

1. **"트랜잭션 이메일을 마케팅 채널로 전환하라."** FedEx의 배송 알림 이메일은 오픈율이 마케팅 이메일의 3~5배에 달한다. 고객이 가장 집중하는 이 순간에 업셀·리뷰 요청·다음 배송 할인을 배치하면 추가 비용 없이 전환을 만든다. → 마케터 적용: 영수증·배송 완료·계정 생성 이메일 등 고오픈율 트랜잭션 이메일에 마케팅 레이어를 추가하라. 단, 정보 밀도는 70:30(트랜잭션:마케팅)을 지켜라.

2. **"파트너 채널 통합이 가장 저마찰 획득 채널이다."** Shopify 앱스토어에 FedEx 앱이 있다는 것은 100만 명의 셀러가 배송사를 비교할 때 FedEx를 기본 선택지로 노출한다는 뜻이다. 배너 광고보다 생태계 내 존재가 낮은 비용으로 더 많은 신규 고객을 만든다. → 적용: 자사 고객이 이미 사용하는 플랫폼(ERP, CRM, e커머스, 회계 소프트웨어)에 자사 서비스를 통합하는 파트너십을 우선 투자하라.

3. **"위기 대응 속도가 브랜드 자산을 결정한다."** 배송 지연을 2시간 내 먼저 알린 FedEx와 고객 컴플레인이 터진 후 24시간 침묵한 FedEx는 같은 서비스 실패를 완전히 다른 브랜드 경험으로 만든다. 위기의 크기가 아니라 응답 속도가 고객 신뢰를 결정한다. → 적용: 서비스 장애·지연 발생 시 고객에게 먼저 알리는 자동화 알림 시스템을 구축하라. 고객이 발견하기 전에 회사가 먼저 말하는 것이 가장 강력한 신뢰 관리다.
