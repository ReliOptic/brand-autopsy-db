# 02. 오디언스 맵 — ServiceNow (NOW)

## 타겟 시장 개요

| 구분 | 내용 |
|------|------|
| 주요 시장 | 대기업(Enterprise) / 중견기업(Mid-Market) |
| 종업원 규모 | 1,000명 이상 (코어 타겟: 5,000명+) |
| 산업 우선순위 | 금융·보험, 헬스케어, 제조, 통신, 공공(정부) |
| 지역 | 북미(~50%), EMEA(~30%), APAC(~20%) (추정) |

## 페르소나 1: CIO / IT 리더십

### 페르소나

**이름**: 김태현 (가상)
**직함**: 최고정보책임자 (CIO)
**소속**: 대형 금융지주사 (임직원 12,000명)
**나이**: 52세
**핵심 KPI**: IT 비용 절감률, 시스템 가용성(SLA), 디지털 전환 로드맵 실행률

**배경**: 20년 IT 경력, 레거시 시스템 유지비가 전체 IT 예산의 60%를 잡아먹는 상황에 직면. 이사회는 "AI 전략"을 요구하고, 현장 직원들은 느린 IT 지원에 불만. 여러 벤더 솔루션이 사일로를 이루고 있어 통합이 시급하다.

**페인 포인트**:
- 이기종 시스템 간 데이터 불일치로 의사결정 지연
- ITIL 프로세스가 있으나 실제 현장 준수율 낮음
- 보안·컴플라이언스 감사 대응에 분기당 수백 시간 낭비

**구매 동기**:
- 단일 플랫폼으로 IT 운영 가시성 확보
- AI 자동화로 IT 헬프데스크 인건비 30% 절감
- 이사회에 제출할 디지털 전환 ROI 수치 확보

**메시지 훅**: "IT 운영 비용을 줄이면서 동시에 비즈니스 속도를 높이는 유일한 방법"

## 페르소나 2: CHRO / HR 디지털화 담당자

### 페르소나

**이름**: 박지은 (가상)
**직함**: 최고인사책임자 (CHRO)
**소속**: 글로벌 제조기업 한국 법인 (임직원 8,000명)
**나이**: 45세
**핵심 KPI**: 직원 만족도(eNPS), 온보딩 완료 시간, HR 티켓 처리 시간

**배경**: 팬데믹 이후 하이브리드 근무 체계를 도입했으나 HR 프로세스는 여전히 이메일 기반. 신입 직원 온보딩에 평균 3주 소요, IT·HR·시설 부서 간 커뮤니케이션 단절이 주요 원인. MZ세대 직원들의 자기서비스 요구 증가.

**페인 포인트**:
- 온보딩 체크리스트가 Excel 파일로 관리되어 책임 소재 불분명
- 직원 문의가 HR 담당자 개인 이메일로 몰려 번아웃 심화
- 퇴직자 처리 시 시스템 접근 권한 회수가 수동으로 진행

**구매 동기**:
- 직원 자기서비스 포털로 HR 문의 60% 자동화
- 온보딩 워크플로우 자동화로 첫날부터 생산적 근무 환경
- 직원 경험 데이터를 통한 이탈 예측 및 선제 대응

**메시지 훅**: "직원이 IT가 아닌 업무에 집중할 수 있는 환경"

## 페르소나 3: IT 운영 관리자 / ServiceDesk 팀장

### 페르소나

**이름**: 이준호 (가상)
**직함**: IT 서비스 운영 팀장
**소속**: 대형 통신사 IT 인프라팀 (팀원 15명)
**나이**: 38세
**핵심 KPI**: MTTR(평균 복구 시간), 1차 해결률(FCR), SLA 준수율

**배경**: 현장에서 직접 인시던트를 다루는 실무 리더. 현재 JIRA·Remedy·Excel을 혼용 중이며 어느 것도 완전하지 않다. 야간 장애 알림이 수동 전화 연락망으로 운영되고, 사후 보고서 작성에 하루를 쓴다. 경영진이 원하는 대시보드를 만들기 위해 별도로 데이터를 정리해야 한다.

**페인 포인트**:
- 인시던트-문제-변경 관리가 분리된 시스템에서 운영되어 컨텍스트 손실
- AI/ML 기반 이상 탐지 없어 장애를 사후에 발견
- 지식 베이스가 오래되어 L1 엔지니어가 동일한 질문을 반복 에스컬레이션

**구매 동기**:
- ITSM + ITOM 통합으로 인시던트-인프라 이벤트 연계 가시성
- AI 기반 티켓 분류·라우팅으로 L1 부담 감소
- 실시간 대시보드로 경영진 보고 자동화

**메시지 훅**: "야간 장애 전화를 줄이고, 팀이 실제 문제 해결에 집중하게"

## 의사결정 구조 (Buying Committee)

```
경제적 구매자 (Economic Buyer)
└── CIO / CFO — 예산 승인권자, ROI·TCO 중심 판단

기술적 구매자 (Technical Buyer)
└── IT 아키텍트 / 운영 팀장 — 기술 적합성·통합 복잡도 평가

사용자 (End User / Champion)
└── IT 서비스 데스크 팀 / HR 오퍼레이션

코치 (Internal Coach)
└── 디지털 전환 TF 담당자 — 내부 변화관리 추진

거부권자 (Blocker)
└── 정보보안 팀장 — 보안·컴플라이언스 우려 관문
```

## 구매 여정 단계별 메시지

| 단계 | 핵심 질문 | ServiceNow 메시지 |
|------|-----------|------------------|
| 인식 (Awareness) | "왜 지금 바꿔야 하나?" | AI 자동화 시대, 수동 워크플로우는 경쟁 열위 |
| 고려 (Consideration) | "우리 환경에 맞는가?" | 업종별 레퍼런스, TCO 계산기, 빠른 PoC |
| 결정 (Decision) | "ROI를 증명할 수 있나?" | 가트너 Magic Quadrant 리더, 고객 성공 사례 |
| 도입 (Onboarding) | "구현이 너무 복잡하지 않나?" | NOW Value Framework, 파트너 에코시스템 |
| 확장 (Expansion) | "다음 유스케이스는?" | 플랫폼 로드맵, 앱 스토어, Creator Workflows |
