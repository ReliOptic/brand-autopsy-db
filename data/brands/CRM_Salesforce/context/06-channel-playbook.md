# 06. 채널 플레이북 해부 — Salesforce (CRM)

## 채널별 전략

### 채널 1 — LinkedIn

| 항목 | 내용 |
|------|------|
| **역할** | B2B 인바운드 리드 생성 + 사고 리더십 포지셔닝 |
| **주요 오디언스** | CRO, CMO, CIO, RevOps Manager, Salesforce Admin |
| **포스팅 빈도** | 주 5–7회 (브랜드 페이지) + 임원 개인 계정 주 2–3회 |
| **최고 성과 콘텐츠 유형** | 고객 성공 사례(동영상/캐러셀), State of Sales 인포그래픽, Agentforce 데모 클립 |
| **광고 형태** | Sponsored Content(단일 이미지/동영상), Lead Gen Form, Thought Leadership Ads |
| **KPI** | MQL 수, CTR(>0.5%), 비디오 시청 완료율(>25%), 팔로워 증가율 |

**링크드인 콘텐츠 공식**:
- 1–2문장 훅 (수치 또는 질문) → 3–5 bullet (가치 포인트) → CTA (링크 또는 리포트 다운로드)
- 이미지 없는 텍스트 전용 포스팅: 임원 개인 계정에서 더 높은 오가닉 도달률

### 채널 2 — Salesforce.com 웹사이트 + SEO

| 항목 | 내용 |
|------|------|
| **역할** | 인바운드 리드 수집 허브 + 제품 정보 허브 |
| **주요 오디언스** | 능동적 CRM 솔루션 탐색자 (Bottom-of-Funnel) |
| **핵심 랜딩 페이지** | 제품별 클라우드 페이지, 업종별 솔루션 페이지, ROI 계산기, 트라이얼 등록 |
| **SEO 전략** | "best CRM" "CRM software" "AI CRM" 등 카테고리 키워드 1위 유지, 비교 키워드("Salesforce vs HubSpot") 공략 |
| **콘텐츠 자산** | 블로그(Salesforce Blog), State of X 연간 리포트, 웨비나 온디맨드, ROI 계산기 |
| **KPI** | 오가닉 세션, 트라이얼 전환율, 랜딩 페이지 리드 CVR |

**핵심 페이지 구조 원칙**: 히어로(문제/솔루션) → 신뢰 배지(Fortune 500 고객 로고) → 제품 데모 영상 → 수치 기반 성과 → 고객 후기 → CTA (데모/트라이얼)

### 채널 3 — Trailhead (자사 학습 플랫폼)

| 항목 | 내용 |
|------|------|
| **역할** | 리드 너처링 + 챔피언 육성 + 커뮤니티 기반 팬덤 구축 |
| **주요 오디언스** | Salesforce Admin 지망생, 현재 고객 사용자, 파트너 개발자 |
| **콘텐츠 형태** | 학습 모듈(Trail), 핸즈온 실습(Trailhead Playground), 자격증 준비 과정, 이벤트(Trailblazer Connect) |
| **게이미피케이션** | 포인트, 배지, 랭크(Ranger/Mountaineer/...) — 완료 동기 유발 |
| **KPI** | 월간 학습자 수, 자격증 취득자 수, 학습→제품 활성화 전환율 |

**전략적 가치**: Trailhead 학습자 1명이 사내 Salesforce 챔피언이 되면 영업 없이 도입이 결정된다. 교육 플랫폼이 곧 영업 채널이다.

### 채널 4 — Dreamforce (연례 컨퍼런스)

| 항목 | 내용 |
|------|------|
| **역할** | 브랜드 종교 행사 + 제품 대규모 런칭 무대 + Trailblazer 정체성 갱신 |
| **주요 오디언스** | 기존 고객 챔피언/임원, 파트너, 미디어, 잠재 고객 |
| **규모** | 연간 17만 명+ (직접 참가), 수백만 명 (온라인 스트리밍) |
| **콘텐츠 형태** | 키노트(CEO + 고객 무대), 300+ 세션, 파트너 엑스포, Trailblazer 어워드 |
| **파생 콘텐츠** | 키노트 하이라이트 영상, 세션 녹화 온디맨드, SNS 하이라이트 |
| **KPI** | 참가자 NPS, 미디어 노출(Earned Media), 행사 중 신규 파이프라인 생성액 |

**운영 원칙**: Dreamforce는 제품 광고가 아니라 "Trailblazer로서의 당신을 축하하는 자리"로 포지셔닝. 고객 스피커를 전체 세션의 40%+ 배치.

### 채널 5 — 파트너 생태계 채널 (AppExchange + SI 파트너)

| 항목 | 내용 |
|------|------|
| **역할** | 리셀러/구현 파트너를 통한 간접 영업 + 세계 최대 B2B 앱 마켓플레이스 |
| **주요 오디언스** | 시스템 통합사(Accenture, Deloitte, Infosys), ISV 파트너, VAR |

| **파트너 인센티브** | 수익 공유, 공동 마케팅(Co-Marketing) 펀드, 파트너 배지 |
| **KPI** | AppExchange 설치 수, 파트너 유래 파이프라인 비율(전체의 70%+ 추정) |

**전략적 가치**: Salesforce 신규 계약의 상당 부분이 구현 파트너의 제안에서 시작된다. 파트너 생태계는 Salesforce의 가장 큰 영업 조직이다.

## 채널 시너지 맵

```
[Trailhead 학습] ──→ [Trailblazer 커뮤니티] ──→ [Dreamforce 참가]
       ↓                      ↓                          ↓
  [내부 챔피언화]        [동료 추천]              [임원 참가 → 구매 결정]
       ↓                      ↓                          ↓
  [Trial 시작]          [AppExchange 탐색]         [파트너 구현 계약]
       ↓                      ↓                          ↓
  [Sales Cloud 도입] ──→ [추가 클라우드 업셀] ──→ [Agentforce 파일럿]
       ↓
  [케이스 스터디 → LinkedIn/Blog] ──→ [Dreamforce 스피커 초청]
```

**핵심 인사이트**: Salesforce의 채널 플라이휠은 "무료 교육(Trailhead) → 커뮤니티 소속감 → 컨퍼런스 경험 → 구매 → 확장 → 고객이 콘텐츠 생산자로 전환"이다. 각 채널이 다음 채널의 트래픽을 공급한다.

## 채널 금지사항

| 금지 행동 | 이유 |
|-----------|------|
| 동일 콘텐츠 전 채널 동시 배포 (Copy-Paste) | 채널별 맥락 무시 → 낮은 인게이지먼트, 알고리즘 불이익 |
| Trailhead에서 유료 제품 업셀 압박 | 교육 공간의 신뢰 훼손 → 학습자 이탈 |
| LinkedIn에서 경쟁사 실명 비판 | 브랜드 격하, 잠재 고객에게 부정적 인상 |
| Twitter/X에서 규제·법적 이슈 즉흥 발언 | 주가 영향 가능성 (상장사 커뮤니케이션 정책 위반) |
| 웨비나에서 확정되지 않은 로드맵 약속 | 고객 기대 관리 실패 → 신뢰 손상 |
| AppExchange 파트너 콘텐츠 미검증 공유 | 파트너 제품 품질 이슈가 Salesforce 브랜드에 귀속 |
| 이메일 캠페인 opt-out 무시 | GDPR/CCPA 위반, 브랜드 신뢰 훼손 |

## 위기 대응 프로토콜

### 위기 유형별 채널 대응

**유형 1 — 플랫폼 장애 / 서비스 중단**
- **1차 대응 채널**: trust.salesforce.com (실시간 상태 페이지) → Twitter/X 공식 계정 업데이트
- **톤**: 투명·사실 기반·책임감 ("현재 상황 / 원인 / 복구 예상 시간 / 사후 조치")
- **금지**: "일부 불편을 드려 죄송합니다" 같은 모호한 사과 → 구체적 영향 범위 공개
- **에스컬레이션**: 4시간 이상 지속 시 CEO/CTO 직접 성명 발표

**유형 2 — 데이터 보안 사고**
- **1차 대응 채널**: 영향 받은 고객 직접 이메일 (SNS보다 선행) → 공식 웹사이트 성명
- **톤**: 법무 검토 완료 후 투명 공개, 피해 범위·조치·보상 명시
- **금지**: SNS에서 먼저 공개하거나 최소화 시도 — 언론 선제 공개가 더 큰 위기 초래
- **에스컬레이션**: 규제 기관 즉시 보고 (GDPR 72시간 이내 의무)

**유형 3 — 임원 발언 논란**
- **1차 대응 채널**: 공식 성명 (Salesforce.com 뉴스룸) → LinkedIn 게시
- **톤**: 명확한 입장 정정, 회사 가치관(Equality, Trust) 재확인
- **금지**: 침묵 또는 모호한 해명 — Salesforce는 사회적 이슈에 명확한 입장을 표명해 온 브랜드
- **에스컬레이션**: Marc Benioff 개인 Twitter/X를 통한 직접 해명 (브랜드 DNA로 정착)

**위기 대응 공통 원칙**:
1. 속도보다 정확도 — 첫 발표가 번복되면 신뢰 이중 손상
2. 채널 우선순위: Trust 페이지 → 직접 이메일 → 공식 웹사이트 → SNS 순
3. "미확인" 상태일 때는 "조사 중"임을 명시하고 업데이트 시점 약속
4. 위기 커뮤니케이션 담당자 1명이 모든 채널 메시지를 조율 (분산 발언 금지)

### Steal Sheet — 3가지

1. **채널 플라이휠을 설계하라 — 단일 채널 의존은 취약하다.** Salesforce의 채널 전략이 강한 이유는 Trailhead → 커뮤니티 → Dreamforce → 케이스 스터디 → LinkedIn으로 이어지는 자기강화 루프 때문이다. 어느 한 채널이 알고리즘 변화나 예산 삭감으로 흔들려도 다른 채널이 버텨준다. 단일 채널(예: 오직 인스타그램)에 의존하는 브랜드는 언제나 플랫폼의 인질이다. 지금 당신의 채널 중 2개가 사라져도 살아남을 수 있는가?

2. **자사 플랫폼(Owned Media)을 가장 크게 키워라.** Salesforce의 Trailhead와 trust.salesforce.com은 어떤 소셜 플랫폼도 차단할 수 없는 자사 소유 채널이다. Instagram이 알고리즘을 바꾸거나 LinkedIn이 도달률을 낮춰도, Trailhead 학습자 수백만 명은 영향 받지 않는다. 마케팅 예산의 일정 비율을 항상 Owned Media(자사 블로그, 학습 플랫폼, 이메일 리스트) 구축에 투자하라.

3. **위기 대응 채널은 평소에 신뢰를 쌓아야 작동한다.** trust.salesforce.com이 장애 발생 시 즉각 신뢰를 얻는 이유는, 장애가 없을 때도 실시간 가동 상태를 투명하게 공개해왔기 때문이다. 위기 때만 등장하는 공식 계정은 아무도 팔로우하지 않는다. 위기 대응 채널은 평상시 운영 루틴에서부터 신뢰를 적립해야 한다.
