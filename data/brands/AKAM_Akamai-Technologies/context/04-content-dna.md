# 04. 콘텐츠 DNA — Akamai Technologies (AKAM)

## 콘텐츠 철학

**핵심 원칙 (추정)**
> "데이터가 먼저, 스토리가 그 다음." — 수치·연구 결과를 앞세우고, 그 위에 비즈니스 임팩트 내러티브를 구축한다.

Akamai 콘텐츠는 세 층위로 구성된다:
1. **인텔리전스 레이어** — State of the Internet 리포트, 위협 연구, 업계 데이터
2. **솔루션 레이어** — 제품 백서, 아키텍처 가이드, 기술 문서
3. **고객 성공 레이어** — 케이스 스터디, ROI 분석, 레퍼런스 아키텍처

---

## 콘텐츠 필라 (Content Pillars)

### 필라 1: 위협 인텔리전스 (Threat Intelligence)
- **목적**: 인식 단계 리드 생성 + 사고 리더십 확립
- **형식**: 연간/분기 리포트, 실시간 위협 대시보드, 블로그 포스트
- **대표 콘텐츠**: State of the Internet / Security 리포트 (공식)
- **KPI**: 다운로드 수, 미디어 인용, 리드 생성

### 필라 2: 기술 교육 (Technical Education)
- **목적**: 평가 단계 신뢰 구축 + SEO
- **형식**: 기술 백서, 개발자 문서, 튜토리얼, 웨비나
- **대표 콘텐츠**: EdgeWorkers 개발자 가이드, API 보안 가이드
- **KPI**: 문서 페이지뷰, 개발자 등록, POC 요청

### 필라 3: 비즈니스 케이스 (Business Case)
- **목적**: 의사결정 단계 경영진 설득
- **형식**: ROI 분석, 고객 사례 연구, Analyst 리포트 요약
- **대표 콘텐츠**: Forrester TEI(총 경제적 임팩트) 분석 (공식)
- **KPI**: 영업 기회 전환율, 계약 규모

### 필라 4: 업계 컴플라이언스 & 규제 (Compliance)
- **목적**: 규제 산업(금융, 의료, 공공) 전문성 증명
- **형식**: 컴플라이언스 가이드, 규제 브리핑, 공동 백서
- **대표 콘텐츠**: PCI-DSS v4.0 대응 가이드, DORA 컴플라이언스
- **KPI**: 해당 산업 리드 비율, RFP 입찰 포함률

## 톤 오브 보이스 가이드라인

**전체 톤**: 권위적이되 독단적이지 않고, 기술적이되 접근 가능하며, 데이터 기반이되 인간적 임팩트를 놓치지 않는다.

| 상황 | 톤 | 예시 표현 |
|------|-----|---------|
| 위협 보고 | 긴박하되 패닉 유발 금지 | "이 공격 패턴은 전년 대비 87% 증가했습니다. 준비된 조직은 대응할 수 있습니다." |
| 제품 설명 | 기술적·정확 | "Prolexic은 9Tbps 이상의 DDoS 공격을 흡수합니다." |
| 고객 성공 | 결과 중심 | "A사는 Akamai 도입 후 DDoS 대응 시간을 72시간에서 2분으로 단축했습니다." |
| 위기/장애 상황 | 투명·침착 | "현재 상황을 파악 중이며 15분마다 업데이트를 제공하겠습니다." |

## Steal Sheet: 레이어별 콘텐츠 전략 3가지

### Steal Sheet Layer 1 — 인식(Awareness) 단계

**SS-A1: 연간 위협 현황 리포트**
- **형식**: 30~50페이지 PDF + 인터랙티브 웹 버전
- **핵심 요소**: 업계별 공격 트렌드 데이터, 지역별 위협 지도, 예측 섹션
- **배포 전략**: PR 엠바고 배포 → 런칭 당일 미디어 동시 공개 → 6개월간 분할 콘텐츠 재가공
- **훔쳐야 하는 이유**: Akamai는 전 세계 트래픽 30% 처리 데이터를 기반으로 다른 벤더가 가질 수 없는 1차 데이터를 보유한다. 이 리포트는 CISO 책상에 오르는 몇 안 되는 벤더 자료다.
- **실행 체크리스트**:
  - [ ] 독점 1차 데이터 확보 (자사 네트워크 로그, 고객 익명 데이터)
  - [ ] 외부 리서치 기관과 공동 발행으로 신뢰도 제고
  - [ ] 업계별 슬라이스 버전 제작 (금융, 미디어, 공공 각각)
  - [ ] 인터랙티브 데이터 시각화 웹페이지 병행

**SS-A2: 실시간 위협 인텔리전스 대시보드**
- **형식**: 공개 웹 대시보드 (무료 접근)
- **핵심 요소**: 실시간 DDoS 공격 지도, 봇 트래픽 비율, 취약점 익스플로잇 트렌드
- **목적**: 미디어 인용 유도 + 브랜드 인지도 상시 유지
- **훔쳐야 하는 이유**: 기자·분석가가 사이버 보안 기사 작성 시 항상 참조하는 자료가 된다. Akamai가 인터넷의 신뢰할 수 있는 척도(barometer)로 포지셔닝된다.

**SS-A3: 업계별 공격 시나리오 시뮬레이터**
- **형식**: 인터랙티브 웹 앱 (무료 체험)
- **핵심 요소**: "당신의 산업에서 가장 흔한 공격은?" → 시나리오별 임팩트 계산 → 솔루션 연결
- **목적**: 리드 캡처 + 제품 인식 연결

---

### Steal Sheet Layer 2 — 고려(Consideration) 단계

**SS-C1: 벤더 비교 기술 가이드**
- **형식**: 15~25페이지 PDF 백서
- **핵심 요소**: 독립적 테스트 기준, 아키텍처 비교, 성능 벤치마크
- **주의**: 경쟁사 직접 명시보다 "업계 선도 CDN 벤더 3사 비교" 형식으로 우회
- **훔쳐야 하는 이유**: 구매 평가 단계의 고객은 비교 자료를 적극 찾는다. Akamai가 먼저 제공하면 평가 기준 자체를 설정할 수 있다.
- **실행 체크리스트**:
  - [ ] Akamai 강점이 드러나는 평가 기준 설계
  - [ ] 제3자 테스트 기관(Miercom, NSS Labs 등) 활용
  - [ ] 게이팅(이메일 캡처) 콘텐츠로 리드 생성

**SS-C2: ROI 계산기 (인터랙티브)**
- **형식**: 웹 기반 계산 도구
- **핵심 요소**: 현재 인프라 비용 입력 → Akamai 도입 후 절감액 계산 → PDF 보고서 다운로드
- **목적**: 경영진 설득 자료를 영업팀이 아닌 고객 스스로 만들게 한다
- **훔쳐야 하는 이유**: Forrester TEI 방법론 기반 계산기는 신뢰도가 높고 구매 위원회에서 사용 가능한 자료가 된다.

**SS-C3: 기술 아키텍처 레퍼런스 시리즈**
- **형식**: 다이어그램 중심 1~2페이지 기술 브리프
- **핵심 요소**: 산업별 (금융, 미디어, 공공) 레퍼런스 아키텍처, 통합 패턴
- **목적**: 아키텍트·엔지니어 레벨 기술 신뢰 구축
- **배포**: 개발자 문서 포털, GitHub, 기술 컨퍼런스 배포

---

### Steal Sheet Layer 3 — 결정(Decision) 및 유지(Retention) 단계

**SS-D1: 고객 성공 케이스 스터디 (숫자 중심)**
- **형식**: 2페이지 PDF + 영상 인터뷰 (1~2분)
- **필수 구조**: 문제(Before) → 솔루션 → 결과(After, 수치 필수)
- **수치 예시**: "DDoS 대응 시간 72시간 → 2분", "CDN 비용 23% 절감", "가용성 99.999% 달성"
- **훔쳐야 하는 이유**: 구매 결정 단계 고객의 90%는 동종업계 레퍼런스를 요구한다. 산업별로 정렬된 케이스 스터디 라이브러리는 영업 사이클을 단축한다.
- **실행 체크리스트**:
  - [ ] 산업별 (금융, 미디어, 공공, 헬스케어, 리테일) 최소 3개씩 확보
  - [ ] 고객 임원 직접 인용 포함
  - [ ] 비디오 + PDF + 1페이지 요약 3종 세트

**SS-D2: QBR(분기 비즈니스 리뷰) 템플릿**
- **형식**: 내부 영업팀 사용 슬라이드 덱
- **핵심 요소**: 분기별 가치 증명 지표, 신규 위협 트렌드, 업셀 기회 식별
- **목적**: 갱신 리텐션 + 계약 확장(Expansion Revenue)
- **훔쳐야 하는 이유**: 고객이 Akamai의 가치를 스스로 인식하게 하는 구조적 접점을 만든다.

**SS-D3: 위협 알림 뉴스레터 (개인화)**
- **형식**: 주간 이메일 (고객 산업·프로필 기반 개인화)
- **핵심 요소**: 해당 고객 산업 위협 동향, Akamai 플랫폼 탐지 통계, 대응 권고
- **목적**: 상시 접점 유지 + 확장 계약 기회 발굴
- **훔쳐야 하는 이유**: 갱신 결정 전 고객이 Akamai를 "단순 벤더"가 아닌 "보안 파트너"로 인식하게 한다.
