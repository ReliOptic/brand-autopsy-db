# 05. Design System Spec — Ameren Corporation (AEE)

> **Disclaimer**: This document is brand strategy analysis based on publicly accessible sources. Source notation: `(official)` = Ameren-published; `(observed)` = direct observation; `(estimated)` = project inference.

---

## 1. Color Palette

| Token Name | HEX | Role | Usage |
|---|---|---|---|
| `color-ameren-blue` | `#005DAA` | Primary Brand Blue | Logo, primary CTAs, headers (observed on ameren.com) |
| `color-ameren-green` | `#00843D` | Clean Energy / Environmental | Sustainability content, clean energy sections (observed) |
| `color-white` | `#FFFFFF` | Background | Page backgrounds (observed) |
| `color-dark` | `#333333` | Primary Text | Body copy (estimated) |
| `color-light-grey` | `#F4F4F4` | Section Background | Alternating sections (observed) |
| `color-alert-orange` | `#FF8C00` | Alert / Outage | Storm warnings, outage notifications (estimated) |
| `color-alert-red` | `#D32F2F` | Emergency | Emergency notifications (estimated) |

### Color Principles
- Blue dominates as the trust color for a utility. Green signals clean energy commitment. Alert colors are functional, not decorative. (T4_INFERRED)

---

## 2. Typography

Standard utility web typography — system sans-serif (Arial/Helvetica stack) for maximum compatibility across the broad demographic customer base. No custom web fonts observed. (estimated)

---

## 3. Layout Principles

- **Outage map is the most important UI element.** During storms, the outage map receives more traffic than any other page. (T4_INFERRED)
- **Bill payment and account management are primary navigation.** Utility websites are functional tools, not brand experiences. (observed on ameren.com)
- **Accessibility is mandatory.** Utilities serve the broadest demographic; WCAG compliance is expected. (T4_INFERRED)

---

## 4. Design Prohibitions

1. **No decorative design elements that interfere with functional utility.** Ameren.com is a service tool first. (T4_INFERRED)
2. **No imagery that trivializes severe weather.** Storm response is serious operational content. (T4_INFERRED)
3. **No complex navigation that delays bill payment access.** (T4_INFERRED)
4. **No brand-heavy design during outage events.** Outage communication prioritizes information, not branding. (T4_INFERRED)
5. **No stock photography that does not represent the actual service territory demographics.** (T4_INFERRED)
6. **No dark mode implementations that reduce readability of critical safety information.** (T4_INFERRED)
