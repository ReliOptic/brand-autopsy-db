# 03. 경쟁 지형도 — Accenture (ACN)

## 경쟁 구도 개요

Accenture는 단일 업종에 속하지 않는다. 전략 컨설팅(McKinsey·BCG·Bain), IT 서비스(IBM·Infosys·TCS·Cognizant), 감사·회계 기반 컨설팅(Deloitte·PwC·EY·KPMG, 이른바 Big4)의 세 전선에서 동시에 경쟁한다. 이 삼중 경쟁 구조가 Accenture 브랜드의 포지셔닝 복잡성을 만든다(추정).

---

## 경쟁자별 분석

### 1. McKinsey & Company
**포지셔닝**: 순수 전략 컨설팅의 절대 권위자. CEO·이사회급 의제 설정.
**강점**: 브랜드 프레스티지 1위, 파트너 1인당 매출 최고 수준, 글로벌 톱 인재 흡인력.
**약점**: 실행(Implementation) 역량 없음 — 보고서 납품 후 책임 종료. 높은 Day-rate로 중견기업 접근 불가.
**Accenture 차별점**: "우리는 전략도 짜고 직접 실행한다." McKinsey가 떠난 자리에서 Accenture가 구현을 맡는 구조가 실제로 자주 발생한다(추정).

### 2. Deloitte (Consulting 부문)
**포지셔닝**: Big4 감사 네트워크 기반의 통합 전문서비스. Deloitte Digital이 디지털 전환 전면에.
**강점**: 감사·세무·법무와의 크로스셀링. 정부·공공 부문 강세. 글로벌 네트워크.
**약점**: 감사 부문과의 이해충돌 가능성. Accenture 대비 기술 구현 인력 규모 열세.
**Accenture 차별점**: 순수 기술 구현 역량의 규모와 깊이. Accenture는 74만+ 인력 중 상당수가 엔지니어·기술 전문가다(공식).

### 3. IBM Consulting
**포지셔닝**: 하드웨어·미들웨어·클라우드(IBM Cloud, Red Hat) 생태계와 연계된 기술 컨설팅.
**강점**: 자체 기술 스택(IBM Watson, Red Hat OpenShift) 보유. 제조·금융 레거시 시스템 경험.
**약점**: IBM 자체 플랫폼 편향 — 멀티벤더 중립성 의문. 성장률 Accenture 대비 정체(추정).
**Accenture 차별점**: 벤더 중립적 멀티클라우드 전략. AWS·Azure·GCP·SAP 등 복수 Alliance를 동시에 보유해 클라이언트 최적 조합을 제안한다(추정).

## 배틀카드

### 배틀카드 1: Accenture vs. McKinsey

| 항목 | Accenture | McKinsey |
|------|-----------|----------|
| 핵심 가치 명제 | 전략 + 실행 통합 | 전략 수립 권위 |
| 실행 역량 | 74만+ 인력, 직접 구현 | 없음 (외부 파트너 연계) |
| 가격대 | 높음 (대규모 계약) | 최고가 (Day-rate 기준) |
| 주요 클라이언트 | 포춘500·정부 | 포춘100·글로벌 기업 CEO |
| 리스크 공유 | Outcome-based 계약 가능(추정) | 납품 후 책임 종료 |
| 콘텐츠 무기 | Technology Vision, Applied Intelligence Report | McKinsey Quarterly, Global Institute |

**세일즈 언어**: "McKinsey가 전략을 그린다면, Accenture는 그 전략을 현실로 만드는 유일한 파트너입니다. 보고서가 서랍에 쌓이는 것을 막으려면 전략과 실행을 한 팀이 담당해야 합니다."

### 배틀카드 2: Accenture vs. Deloitte

| 항목 | Accenture | Deloitte |
|------|-----------|----------|
| 핵심 가치 명제 | 기술 구현 주도 디지털 전환 | 감사·세무·컨설팅 통합 서비스 |
| 기술 인력 규모 | 전체 74만+ 중 기술 비중 높음 | Deloitte Digital 별도 운영 |
| 산업 특화 | 19개 산업 Practice 운영 | 감사 클라이언트 기반 산업 |
| AI/Cloud 역량 | AWS·Azure·GCP 트리플 Alliance | 일부 플랫폼 Alliance |
| 이해충돌 리스크 | 없음 (비감사 법인) | 감사 법인과 컨설팅 경계 관리 필요 |
| 글로벌 일관성 | 단일 법인 구조로 글로벌 일관성 높음 | 파트너십 구조로 국가별 편차 존재(추정) |

**세일즈 언어**: "Deloitte가 기존 감사 관계를 기반으로 접근한다면, Accenture는 순수한 디지털 전환 성과만으로 선택받습니다. 이해충돌 없이, 기술로만 승부합니다."

### 배틀카드 3: Accenture vs. Infosys / TCS (인도계 IT 서비스)

| 항목 | Accenture | Infosys / TCS |
|------|-----------|---------------|
| 핵심 가치 명제 | 전략 주도 디지털 전환 | 비용 효율적 IT 아웃소싱·운영 |
| 가격 경쟁력 | 프리미엄 가격 | 아웃소싱 특화 저비용 |
| 전략 역량 | 강함 (컨설팅 Practice 보유) | 약함 (구현·운영 특화) |
| 브랜드 포지션 | "변환 파트너" | "효율화 파트너" |
| AI·혁신 주도 | Accenture Research·AI 전략 | 추격자 포지션(추정) |
| C-Suite 접근성 | CEO·CFO 레벨 의사결정 참여 | CIO·IT 운영 레벨 중심 |

**세일즈 언어**: "비용 절감이 목표라면 인도계 IT 서비스로 충분합니다. 하지만 비즈니스 모델을 재창조하고 경쟁 우위를 만드는 것이 목표라면, Accenture가 유일한 선택입니다."

## 포지셔닝 맵

```
                    고전략성 (전략 컨설팅)
                           |
              McKinsey  BCG|Bain
                           |
  낮은 실행력 ←------------+------------→ 높은 실행력
                           |    * Accenture (최상단 우측)
              Big4         |         IBM Consulting
           (Deloitte 등)   |
                           |     Infosys / TCS / Cognizant
                    저전략성 (IT 운영·아웃소싱)
```

Accenture는 전략성과 실행력 모두에서 중상위권을 점유하는 유일한 포지션을 목표로 한다(추정). 이는 "Blue Space" — 아무도 완전히 점유하지 못한 전략-실행 통합 지대다.

### Steal Sheet — 3가지

1. **"경쟁자의 약점을 역으로 브랜드 자산화"** — Accenture는 McKinsey의 "실행 부재"와 인도계 IT의 "전략 부재"를 동시에 공격해 가운데 포지션을 정당화한다. 훔칠 구조: 자사 카테고리에서 좌측 경쟁자의 약점(예: 비싸지만 결과 없음)과 우측 경쟁자의 약점(예: 싸지만 품질 없음)을 모두 공격해 "Goldilocks Position" 확보.

2. **배틀카드를 영업팀 언어로 내재화** — Accenture 어카운트 매니저는 경쟁사가 언급될 때 즉각 사용할 수 있는 표준화된 반박 언어를 훈련받는다(추정). 훔칠 구조: 3대 경쟁자별 "1문장 차별화 클레임 + 1개 수치 근거"를 Sales Playbook에 포함.

3. **"Alliance = 중립성 증명"** — 경쟁사(특히 IBM)의 자체 플랫폼 편향과 달리, Accenture는 복수의 상호 경쟁하는 플랫폼 벤더와 동시에 Alliance를 맺음으로써 중립성을 증명한다. 훔칠 구조: 자사 솔루션이 특정 벤더에 종속되지 않음을 "멀티 파트너십 생태계"로 시각화해 클라이언트의 Lock-in 우려를 선제적으로 해소.
