# 02. 타겟 오디언스 지형도 — Datadog (DDOG)

## ICP (Ideal Customer Profile)

Datadog의 ICP는 '모니터링 툴이 필요한 기업'이 아니라 '클라우드 네이티브 전환을 진행 중이거나 완료한, 100명 이상 엔지니어를 보유한 성장 단계 기업'이다. 구매 결정은 DevOps 엔지니어·SRE가 Bottom-up으로 발의하고, CTO·VP of Engineering이 Top-down으로 확정하는 이중 구조를 가진다.

| 항목 | 정의 |
|------|------|
| 기업 규모 | 핵심: 시리즈 B 이상 스타트업 ~ 글로벌 엔터프라이즈. 엔지니어 50명 이상이 실질 활성 구간. |
| 산업 | 핀테크·이커머스·SaaS·미디어·헬스테크·게임·클라우드 인프라. 공통점은 '가용성 1분 다운타임 = 직접 매출 손실'인 구조. |
| 지역 | 미국(60%+), 유럽(20%), APAC(15%). 한국: 쿠팡·카카오·네이버·토스 등 대형 테크 기업 도입 + 핀테크 스타트업 확산 추세. (추정) |
| 기술 스택 | AWS / GCP / Azure 중 멀티클라우드. Kubernetes·Docker 운영. MSA(마이크로서비스 아키텍처) 전환 중 혹은 완료. |
| 구매 성숙도 | Naive(처음 모니터링 도입) < **Aware(기존 툴 불만족)** < **Informed(Datadog 비교 중)** < Power User(멀티 제품 사용). |
| 도입 트리거 | ① 장애 사후(post-mortem) 이후 "다시는 이런 일이 없어야" ② 클라우드 마이그레이션 ③ 엔지니어링 팀 10명 이상 성장 ④ 기존 툴(Nagios·Zabbix) 한계 체감 |
| 예산 권한 | 월 $1,000~수십만 달러. 연간 계약(Annual Commit) 비율 높음. CTO/VP Eng 최종 승인, 하지만 POC는 IC 엔지니어가 주도. |

## 구매 여정별 고객 질문

Datadog의 구매 여정은 'B2B Bottom-up + Top-down 이중 경로'다. 엔지니어가 먼저 써보고, 관리자가 예산을 확정한다.

| 단계 | 고객이 묻는 질문 | 감정 상태 | Datadog이 제공하는 콘텐츠 |
|------|-----------------|-----------|--------------------------|
| Awareness | "Datadog이 Prometheus보다 뭐가 낫나?" | 호기심+회의 | 기술 블로그, HN 스레드, 컨퍼런스 발표(KubeCon 등) |
| Trial (14일 무료) | "우리 스택에 실제로 붙나?" | 기대+불안 | Quick Start 가이드, 통합 문서, Slack 커뮤니티 |
| PoC (30일) | "팀 전체가 쓸 수 있나? ROI가 나오나?" | 검증+압박 | 솔루션 엔지니어 지원, 고객 사례 레퍼런스, ROI 계산기 |
| 구매 결정 | "계약 조건·보안 인증·SLA가 기준에 맞나?" | 협상+위험관리 | 엔터프라이즈 계약팀, SOC2/ISO27001 인증서, DPA 문서 |
| 확장 (Expand) | "APM 쓰는데 로그도 붙일까? 보안도?" | 신뢰+관성 | 플랫폼 데모, 사용량 대시보드, CSM(고객 성공 팀) |
| 옹호 (Advocacy) | "다른 팀·회사에 추천할 수 있나?" | 자부심+소속감 | DASH 컨퍼런스 스피커 초청, 고객 사례 공동 발표 |
